# 生产发布前检查清单

这份清单只针对当前仓库的 Kubernetes 发布方式，目标是把 `deploy/deploy.sh` 的人为失误风险压到最低。

## 发布原则

- 只使用 `deploy/deploy.sh` 作为生产配置入口。
- 不要直接把 `deploy/k8s/configmap.yaml` 或 `deploy/k8s/keycloak/configmap.yaml` 整体 apply 到现网。
- 发布完成后必须跑一次 `./scripts/validate-k8s-deploy.sh`。

## 发布前

1. 确认当前分支和待发布 commit 正确，工作区没有会误带入的本地改动。
2. 确认 `kubectl` context 指向目标生产集群，而不是 stage / 本地集群。
3. 确认 cloudflared / DNS / TLS 已提前准备好这 3 个公网地址：
   - `AUTH9_PORTAL_URL`
   - `AUTH9_CORE_PUBLIC_URL`
   - `KEYCLOAK_PUBLIC_URL`
4. 确认 `auth9-secrets` 中的以下值已准备好或允许脚本生成：
   - `DATABASE_URL`
   - `REDIS_URL`
   - `JWT_SECRET`
   - `JWT_PRIVATE_KEY`
   - `JWT_PUBLIC_KEY`
   - `SESSION_SECRET`
   - `SETTINGS_ENCRYPTION_KEY`
   - `PASSWORD_RESET_HMAC_KEY`
   - `AUTH9_ADMIN_PASSWORD`
   - `AUTH9_ADMIN_EMAIL`
   - `KEYCLOAK_ADMIN_CLIENT_SECRET`（若重建初始化链路则可由 init job 回填）
5. 明确生产是否需要 tracing：
   - 如果不需要，保持 `OTEL_TRACING_ENABLED=false`
   - 如果需要，先准备真实 OTLP endpoint，再打开 tracing
6. 明确生产 branding 外链域名。
   - 如果要加载外部 logo / favicon，先设置好 `BRANDING_ALLOWED_DOMAINS`

## 发布中

1. 执行 `./deploy/deploy.sh`
2. 记录脚本打印的管理员凭据区块，特别是首次生成的 `AUTH9_ADMIN_PASSWORD`
3. 确认脚本结束时没有 rollout timeout、init job failure、client secret patch failure

## 发布后

1. 立即执行：

```bash
./scripts/validate-k8s-deploy.sh --namespace auth9
```

2. 至少人工确认 1 次登录主链路：
   - 打开 Portal 登录页
   - 点击“使用密码登录”
   - 确认跳转 Keycloak
   - 使用管理员账号登录
   - 成功进入租户选择或 Dashboard
3. 检查最近日志中是否有持续刷新的基础设施错误：

```bash
kubectl logs deploy/auth9-core -n auth9 --since=5m --tail=200
kubectl logs deploy/keycloak -n auth9 --since=5m --tail=200
```

4. 如果发布后需要升级镜像，只使用 `deploy/upgrade.sh` 做 rollout，不要回退到手工 apply 示例 manifest

## 当前重点防坑项

- `APP_BASE_URL` 必须等于 `AUTH9_PORTAL_URL`
- `JWT_ISSUER` 必须等于 `AUTH9_CORE_PUBLIC_URL`
- `keycloak-config.AUTH9_API_URL` 必须等于 `AUTH9_CORE_PUBLIC_URL`
- `keycloak-config.KC_HOSTNAME` 必须等于 `KEYCLOAK_PUBLIC_URL`
- `auth9-portal` 不应整包注入 `auth9-secrets`
- `AUTH9_ADMIN_PASSWORD` 必须稳定持久化，不能在重复部署时被意外轮换

## 建议命令

```bash
# 1. 交互式部署
./deploy/deploy.sh

# 2. 部署后自动校验
./scripts/validate-k8s-deploy.sh --namespace auth9

# 3. 发布后抽查 rollout
kubectl get pods -n auth9
kubectl get configmap auth9-config -n auth9 -o yaml
kubectl get configmap keycloak-config -n auth9 -o yaml
```
